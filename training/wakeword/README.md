# "hey lafufu" Wake-Word Training

This directory is the staging area for training a custom **"hey lafufu"** wake-word
model. Once trained, the resulting model is dropped into `assets/wakeword/` and
loaded by the agent's mic gate.

> **Nothing in this directory other than this README is committed.** The training
> repo, dataset, and intermediate model artifacts are gitignored (see the root
> `.gitignore`). Only the final `.onnx` ends up in `assets/wakeword/`.

## Goal

Replace the stock `hey_jarvis_v0.1` model used by
`packages/agent/src/lafufu_agent/wakeword.py` (`OpenWakeWordDetector`) with a
model that fires on **"hey lafufu"** and stays quiet on everything else.

`OpenWakeWordDetector.__init__` already accepts either a built-in name *or* a
path to an `.onnx` / `.tflite` file, so wiring the trained model in is a config
change — no code change. The agent picks it up via `LAFUFU_WAKEWORD_MODEL`.

## Where the trained model goes

Final artifact path (this is what the agent looks for):

```
assets/wakeword/lafufu.onnx
```

Point the agent at it with:

```
LAFUFU_WAKEWORD_MODEL=assets/wakeword/lafufu.onnx
```

## Recommended pipeline: atlas-voice-training

**Heads-up:** openwakeword's *official* Colab notebook is broken as of May 2026
(see upstream issues
[#296](https://github.com/dscripka/openWakeWord/issues/296),
[#317](https://github.com/dscripka/openWakeWord/issues/317),
[#326](https://github.com/dscripka/openWakeWord/issues/326)). The trainer pins
`torch==1.13.1` which requires Python ≤3.11, plus there are
`pyarrow` / `fsspec` / `webrtcvad` version conflicts. Don't waste time on the
upstream notebook.

Use the community-maintained Dockerized pipeline instead:
**[briankelley/atlas-voice-training](https://github.com/briankelley/atlas-voice-training)**.
It freezes the entire dependency stack to a known-working set and ships an
interactive wrapper.

### Steps

1. **Clone atlas-voice-training into this directory** (it's gitignored):

   ```bash
   cd training/wakeword
   git clone https://github.com/briankelley/atlas-voice-training.git
   cd atlas-voice-training
   ```

2. **Follow its Docker workflow.** Target phrase: `hey lafufu`.

3. **Inputs the pipeline pulls:**
   - **Positives:** synthetic samples generated by `piper-sample-generator`
     across many TTS voices, speaking rates, and pitches.
   - **Negatives:** ~20 GB of background / speech audio (Common Voice,
     AudioSet noise, etc.) — the pipeline scripts download these.

4. **Output:** a single `.tflite` / `.onnx` file, ~1–2 MB.

5. **Copy the result** to the committed asset dir:

   ```bash
   cp <output>.onnx ../../../assets/wakeword/lafufu.onnx
   ```

## Smoke-test the trained model

Once `assets/wakeword/lafufu.onnx` exists:

```bash
uv run python scripts/demo_wakeword.py --model assets/wakeword/lafufu.onnx
```

`--model` accepts either a built-in openwakeword name or a path to a local
model file. Speak "hey lafufu" — the confidence bar should spike past the
threshold; ambient speech and other phrases should stay low.

If it triggers cleanly on the dev box, set
`LAFUFU_WAKEWORD_MODEL=assets/wakeword/lafufu.onnx` in the agent's env and
ship it.
